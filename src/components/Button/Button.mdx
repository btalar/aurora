## Responsiveness

The Button component is fully responsive and adapts to different screen sizes using a mobile-first approach.

### Breakpoint Behavior

| Breakpoint | Width | Button Behavior |
|------------|-------|-----------------|
| **Mobile** | ‚â§640px | Full-width buttons with larger touch targets (min 44px height) |
| **Small** | ‚â•640px | Auto-width with minimum width constraints |
| **Tablet** | ‚â•768px | Increased spacing and padding |
| **Desktop** | ‚â•1024px | Enhanced hover effects with elevation and transforms |
| **XL** | ‚â•1250px | Larger font sizes for large buttons |

### Mobile Optimizations

- **Full-width layout** on mobile (‚â§640px) for easier tapping
- **Minimum touch targets** of 44px (iOS) to 52px for better accessibility
- **Reduced icon gaps** on small screens for better visual balance
- **No hover effects** on touch devices (uses `@media (hover: hover)`)

### Testing Responsiveness

Use the Storybook viewport addon (toolbar icon) to test buttons at different screen sizes:
1. Select different viewports from the dropdown
2. Observe how buttons adapt their size, padding, and layout
3. On mobile viewport, buttons become full-width
4. On desktop viewport with mouse, see enhanced hover effects
